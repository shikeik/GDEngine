# 生命周期 (Lifecycle)

当您编写脚本或组件时，了解执行顺序非常重要。

## 流程图
1. **Awake:** 组件被添加 (`addComponent`) 时立即执行。用于初始化自身变量。
2. **Start:** 游戏进入 Play 模式的第一帧执行。用于获取其他组件引用或初始化跨物体逻辑。
3. **FixedUpdate:** 固定时间步长更新 (默认 60次/秒)，用于物理计算。
4. **Update:** 每帧渲染前执行，用于逻辑判断、输入处理。
5. **Render:** 渲染循环。
6. **Destroy:** 物体被销毁时触发。

## 模式区别
*   **EDIT 模式:** 默认不执行 Update，除非组件类标记了 `@ExecuteInEditMode`。
*   **PLAY 模式:** 执行完整的生命周期。
